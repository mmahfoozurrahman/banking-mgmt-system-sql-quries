*) Find three (3) employees who have the second highest salary from the employee table.

SELECT * FROM employee 
WHERE salary = (
    SELECT DISTINCT salary 
    FROM employee 
    ORDER BY salary DESC 
    LIMIT 1 OFFSET 1
)
LIMIT 3;

*) Find the second highest salary from the employee table using a subquery.

SELECT MAX(salary) AS Second_Highest_Salary
FROM employee
WHERE salary < (SELECT MAX(salary) FROM employee);

*) Find the pending balance from the account table where the result satisfies the following conditions. 
These four conditions must be implemented in a single query:

a. Show the average pending balance  
b. The account status should be active  
c. Group the results by OPEN_BRANCH_ID  
d. Show only those branches where the pending balance is below 4300

SELECT OPEN_BRANCH_ID, AVG(PENDING_BALANCE) AS Average_Pending_Balance
FROM account
WHERE STATUS = 'ACTIVE'
GROUP BY OPEN_BRANCH_ID
HAVING Average_Pending_Balance < 4300;

*) Write a pagination query and explain how it works.

If we need to split large dataset to smaller dataset and display that to page we need to use Pagination (e.g., showing 10 results at a time from total of 100 results).

SELECT * FROM account 
ORDER BY ACCOUNT_ID ASC 
LIMIT 10 OFFSET 20;

So, this is how it works

ORDER BY: ensures consistent order for pagiantion

LIMIT: Specifies the number of records to return that is page size (In this case, 10 records).

OFFSET: Specifies how many records to skip before starting to return rows. Here an offset of 20 means we are skipping the first two pages (as the page size is 10) and starting result set from the 21th record.
